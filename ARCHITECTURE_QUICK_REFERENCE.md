## L2 语义过滤层 (待实现)

### 核心目标
过滤 40-60% 琐事，减少不必要的 L4 调用

### 判定规则
```python
是否琐事 = 重要性评分 < 0.5

重要性评分 ∈ [0, 1]
├─ 0.0-0.2: 环境噪音 (鸟叫、落叶)
├─ 0.3-0.5: 随意交互 (闲聊、问候)
├─ 0.6-0.8: 常规事件 (工作、任务)
└─ 0.9-1.0: 重大事件 (战斗、死亡、背叛)
```

## L3 初步规划层 (待实现)

### 核心目标
快速生成 3-5 个候选方案，供 L4 选择优化

### 工作流
```
事件输入
  ↓
L3 生成 3 个初步方案 (100 Tokens)
  ├─ 方案 A: "逃离" (置信度 85%)
  ├─ 方案 B: "战斗" (置信度 60%)
  └─ 方案 C: "谈判" (置信度 70%)
  ↓
L4 选择最优方案进行深度优化 (200 Tokens)
  ├─ 方案 A 深化 → 5 步计划
  ├─ 跳过 B (低置信度)
  └─ 跳过 C (次优)
  ↓
最终执行
```

### Prompt 模板
```markdown
**任务**: 快速列举 3 个可能的应对方案

**背景**:
- 事件: {event_description}
- NPC 状态: 能量={energy}, 愤怒={anger}
- 相关记忆: {top_2_memories}

**要求**:
- 每个方案 20 字以内
- 标注置信度 (0-1)
- 估计 Token 成本

**回复格式**:
1. [方案名] | 置信度: 0.X | 成本: ~YY Tokens
2. ...
3. ...
```

### 集成示例
```python
# L3 生成方案
plans = await planner.draft_plans(event, npc_state)

# L4 选择并优化
best_plan = plans[0]  # 最高置信度
detailed_plan = await react_agent.optimize(best_plan)
```

### 根据 Token 预算选择策略


## 常见问题解决

### Q1: 如何判断是否需要 L2 过滤?
**A**: 检查 Token 消耗
- 若 > 500 Token/决策 → 需要 L2
- 若有 > 40% 琐事被进行深度推理 → 需要 L2

### Q2: L2 过滤的精度不够，如何改进?
**A**: 选项
1. 增加琐事示例库 (效果 +5%)
2. 调整相似度阈值 (threshold 0.7 → 0.8)
3. 升级到更大的模型 (all-mpnet-base-v2)

### Q3: L3 规划总是生成不了好的候选方案?
**A**: 优化 Prompt
- 提供更清晰的 NPC 状态描述
- 包含更多上下文 (相关记忆, 历史决策)
- 明确指定输出格式

### Q4: L4 推理超时怎么办?
**A**: 限制推理深度
```python
# 当前: max_steps = 10
# 改为: max_steps = 5 (快速模式)
# 或:   max_steps = 3 (超快速模式)

if response_time > 5.0:
    max_steps = min(max_steps - 1, 3)
```
